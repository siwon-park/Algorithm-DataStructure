# 트라이(Trie)

## 개념

> 효율적인 문자열 저장 및 탐색을 위한 자료 구조

여러 문자열을 저장하는 자료 구조로 `레딕스 트리(radix tree)` 또는 `접두사 트리(prefix tree)`라고도 함

### 성능

문자열 삽입 및 탐색: `O(M)`

#### 성능 비교

문자열의 최대 길이: `M`, 문자열의 개수: `N`

- 단순 비교 시: 최악의 경우 `O(MN)`
- 이진 탐색 시: 최악의 경우 `O(NMlogN)`
  - 탐색: `O(MlogN)` → 최악의 경우 N개까지 탐색
